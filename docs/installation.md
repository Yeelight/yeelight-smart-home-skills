# Installation

[简体中文](installation.zh-CN.md)

## 1. Install Yeelight Home

Both Skills require the local `yeelight-home` Runtime.

### Homebrew

```sh
brew install Yeelight/tap/yeelight-home
```

### npm

```sh
npm install -g yeelight-home
```

### GitHub Release

Download the package for your operating system from [Yeelight Home Releases](https://github.com/Yeelight/yeelight-home/releases), then place `yeelight-home` on `PATH`.

Verify the Runtime and sign in without sharing credentials with an AI assistant:

```sh
yeelight-home --version
yeelight-home doctor --json
yeelight-home auth login --qr
yeelight-home auth status --json
```

## 2. Install The Skills

### skills.sh

```sh
npx skills add https://github.com/Yeelight/yeelight-smart-home-skills --skill yeelight-smart-home
npx skills add https://github.com/Yeelight/yeelight-smart-home-skills --skill yeelight-pro-app-builder
```

Install only the Skill you need, or run both commands.

### OpenClaw / ClawHub

The optional ClawHub channel currently covers the direct-control Skill:

```sh
openclaw skills install @yeelight/yeelight-smart-home
```

### Manual installation

Clone this repository and copy the selected directory under `skills/` into your agent host's Skill directory. Keep the directory name unchanged so it matches the `name` in `SKILL.md`.

```sh
git clone https://github.com/Yeelight/yeelight-smart-home-skills.git
```

Consult your host documentation for its Skill directory. Common hosts can also discover the repository directly through skills.sh.

## 3. Verify

Restart the agent host after installation, then ask it to identify the installed Skill before requesting a device operation.

For direct control, verify the Runtime separately:

```sh
yeelight-home doctor --json --online
```

For Builder use, confirm Node.js 22 or later:

```sh
node --version
```

## Upgrade

Repeat the skills.sh installation command to refresh the installed copy. Upgrade the Runtime independently with the package manager used during installation.

## Security

- Never paste a token, password, cookie, or QR login result into chat.
- The Skills call the local Runtime; browser applications generated by Builder use a loopback Bridge.
- Persistent or sensitive changes remain subject to Runtime validation and confirmation.
